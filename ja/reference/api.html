公開 API¶

PME 2.0 以降、import pme で安定した公開 API にアクセスできます。

import pme

# コード実行
result = pme.execute("print(C.mode)")

# メニュー操作
pm = pme.find_pm("My Pie Menu")
if pm:
    pme.invoke_pm(pm)

実行 API¶

pme.execute(code, *, extra_globals=None)¶

任意の Python コードを PME の標準名前空間で実行します。

パラメータ:

  • code (str) -- 実行する Python コード（複数行可）

  • extra_globals (dict) -- 追加のグローバル変数（オプション）

戻り値:

実行結果を含む ExecuteResult

戻り値の型:

ExecuteResult

例:

result = pme.execute("print(C.mode)")
if not result.success:
    print(f"Error: {result.error_message}")

# 追加変数を渡す
result = pme.execute(
    "print(gizmo.name)",
    extra_globals={"gizmo": my_gizmo}
)

注釈

実行時に利用可能な標準変数: C, D, bpy, E, L, U, delta, drag_x, drag_y など

pme.evaluate(expr, *, extra_globals=None)¶

式を評価して結果を返します。

パラメータ:

  • expr (str) -- 評価する Python 式

  • extra_globals (dict) -- 追加のグローバル変数（オプション）

戻り値:

式の評価結果

Raises:

構文エラーや未定義変数の場合は例外を送出

例:

mode = pme.evaluate("C.mode")
is_edit = pme.evaluate("C.mode == 'EDIT_MESH'")

警告

execute() と異なり、失敗時は例外を送出します。 フォールバックが必要な場合は try-except で囲んでください。

class pme.ExecuteResult¶

実行結果を表すデータクラス。

success: bool¶

実行が成功したかどうか

error_message: str | None¶

失敗時のエラーメッセージ

メニュー API¶

pme.find_pm(name)¶

名前でパイメニューを検索します。

パラメータ:

name (str) -- メニューの正確な名前

戻り値:

見つかった場合は PMHandle、それ以外は None

例:

pm = pme.find_pm("My Pie Menu")
if pm:
    print(f"Found: {pm.name} ({pm.mode})")
pme.list_pms(mode=None)¶

すべてのパイメニューを列挙します。

パラメータ:

mode (str) -- メニュータイプでフィルタ（'PMENU', 'RMENU', 'DIALOG' など）

戻り値:

PMHandle のリスト

例:

all_menus = pme.list_pms()
pie_menus = pme.list_pms(mode='PMENU')
pme.invoke_pm(pm_or_name)¶

パイメニューを呼び出し（表示）します。

パラメータ:

pm_or_name (PMHandle | str) -- PMHandle またはメニュー名

戻り値:

成功した場合は True

例:

pme.invoke_pm("My Pie Menu")

# または
pm = pme.find_pm("My Pie Menu")
if pm:
    pme.invoke_pm(pm)
class pme.PMHandle¶

パイメニューへのハンドル。

name: str¶

メニュー名

mode: str | None¶

メニュータイプ（'PMENU', 'RMENU', 'DIALOG' など）

enabled: bool¶

メニューが有効かどうか

スキーマ API¶

pme.schema¶

PME スキーマレジストリのインスタンス。メニュー定義のパース・管理に使用。

注釈

pme.props は後方互換のためのエイリアスです。新規コードは pme.schema を使用してください。

スクリプト名前空間¶

PME のスロットエディタ（コマンドタブ、カスタムタブ）内で利用できる変数と関数です。

変数

bpy.context へのアクセス

D

bpy.data へのアクセス

bpy

Blender Python モジュール

変数

説明

icon

現在のアイテムのアイコン名

現在のアイテムのアイコン値（整数）

ユーザーデータ¶

U¶

セッション内で持続するユーザーデータストレージ。

使用例:

# 値の設定
U.my_counter = 0
U.update(foo="value1", bar="value2")

# 値の取得
value = U.my_counter
value = U.get("my_counter", 0)  # デフォルト値付き

注釈

Blender を再起動するとデータは失われます。

グローバル関数¶

メニュー操作¶

open_menu(name, slot=None, **kwargs)¶

名前でメニューを開きます。

パラメータ:

  • name (str) -- メニュー名

  • slot (int | str | None) -- スロットのインデックスまたは名前（スタックキー用）

  • kwargs -- モーダル/マクロオペレーターに渡す引数

戻り値:

メニューが存在する場合は True

例:

# 条件に応じてメニューを開く
open_menu("Lamp Menu" if C.active_object.type == 'LAMP' else "Object Menu")

# スタックキーのスロット指定
open_menu("My Stack Key", "Ctrl slot" if E.ctrl else "Default slot")
draw_menu(name, frame=False, dx=0, dy=0, layout=None)¶

別のポップアップダイアログ内にメニューを描画します。

パラメータ:

  • name (str) -- メニュー（ポップアップダイアログ）の名前

  • frame (bool) -- フレームで囲むかどうか

  • dx (int) -- 水平オフセット

  • dy (int) -- 垂直オフセット

  • layout -- カスタムレイアウト（オプション）

戻り値:

メニューが存在する場合は True

toggle_menu(name, value=None)¶

メニューを有効/無効にします。

パラメータ:

  • name (str) -- メニュー名

  • value (bool) -- True で有効、False で無効、None でトグル

戻り値:

メニューが存在する場合は True

スクリプト実行¶

execute_script(path, **kwargs)¶

外部 Python スクリプトを実行します。

パラメータ:

  • path (str) -- スクリプトファイルパス（相対パスは pie_menu_editor フォルダから）

  • kwargs -- スクリプトに渡すキーワード引数

戻り値:

スクリプト内の return_value または True

スクリプト内で利用可能な変数: kwargs, __file__, return_value, PME のすべてのグローバル変数

例:

# 基本的な実行
execute_script("scripts/hello_world.py", msg="Hello!")

# 戻り値の取得
result = execute_script("scripts/get_data.py")

警告

信頼できるソースのスクリプトのみ実行してください。

プロパティ操作¶

props(name=None, value=None)¶

PME プロパティの値を取得または設定します。

パラメータ:

  • name (str) -- プロパティ名（None でコンテナを返す）

  • value -- 設定する値

戻り値:

コンテナ、プロパティ値、または True

例:

# 値の取得
value = props("MyProperty")
value = props().MyProperty

# 値の設定
props("MyProperty", new_value)
props().MyProperty = new_value

UI ヘルパー¶

message_box(text, icon='INFO', title='Pie Menu Editor')¶

メッセージボックスを表示します。

パラメータ:

  • text (str) -- 表示するメッセージ（\n で改行）

  • icon (str) -- アイコン名（'INFO', 'ERROR', 'QUESTION' など）

  • title (str) -- ウィンドウタイトル

戻り値:

True

例:

message_box("処理が完了しました", icon='INFO')
message_box("エラーが発生しました\n詳細を確認してください", icon='ERROR')
overlay(text, **kwargs)¶

オーバーレイメッセージを描画します。

パラメータ:

  • text (str) -- 表示するメッセージ

  • kwargs --

    • alignment: 'TOP', 'TOP_LEFT', 'TOP_RIGHT', 'BOTTOM', 'BOTTOM_LEFT', 'BOTTOM_RIGHT'

    • duration: 表示時間（秒、デフォルト 2.0）

    • offset_x, offset_y: オフセット（ピクセル）

戻り値:

True

例:

overlay("Hello PME!", duration=1.0, offset_y=100)
input_box(func=None, prop=None)¶

入力ボックスを表示します。

パラメータ:

  • func -- 入力値で呼び出すコールバック関数

  • prop (str) -- 編集するプロパティパス

戻り値:

True

例:

# プロパティを直接編集
input_box(prop="C.active_object.name")

# コールバックで処理
input_box(func=lambda value: overlay(f"入力: {value}"))
close_popups()¶

すべてのポップアップダイアログを閉じます。

戻り値:

True

tag_redraw(area=None, region=None)¶

UI エリアを再描画します。

パラメータ:

  • area (str) -- 再描画する Area タイプ（None で全エリア）

  • region (str) -- 再描画する Region タイプ（None で全リージョン）

戻り値:

True

カスタムタブ専用¶

panel(pt, frame=True, header=True, expand=None, poll=True, layout=None)¶

パネルを描画します。

パラメータ:

  • pt (type | str) -- パネルクラスまたはクラス名

  • frame (bool) -- フレームで囲むかどうか

  • header (bool) -- ヘッダーを表示するかどうか

  • expand (bool | None) -- 展開状態（True/False/None で前回の状態）

  • poll (bool) -- poll メソッドを実行するかどうか

  • layout -- カスタムレイアウト

戻り値:

True

例:

panel("MATERIAL_PT_context_material", True, True, True)
operator(layout, operator, text='', icon='NONE', emboss=True, icon_value=0, **kwargs)¶

プロパティ付きでオペレーターボタンを描画します。

パラメータ:

  • layout -- UILayout インスタンス

  • operator (str) -- オペレーター識別子

  • text (str) -- ボタンテキスト

  • icon (str) -- アイコン名

  • kwargs -- オペレータープロパティ

戻り値:

OperatorProperties オブジェクト

例:

operator(L, "wm.context_set_int", "Material Slot 1",
        data_path="active_object.active_material_index", value=0)
custom_icon(filename)¶

カスタムアイコンの整数値を取得します。

パラメータ:

filename (str) -- pie_menu_editor/icons/ 内の拡張子なしファイル名

戻り値:

アイコン値（整数）

例:

L.label(text="Custom Icon", icon_value=custom_icon("my_icon"))

UserData 拡張（計画中）¶

現在の U は単純で使いやすいですが、大規模なスクリプトでは管理が難しくなります。 以下の拡張を検討中です。

名前付きインスタンス¶

# 将来の API 案
my_data = pme.create_user_data("my_addon")
my_data.counter = 0

# グローバル U は従来通り
U.temp = "global"

レジストリ¶

# 将来の API 案
# 登録されたインスタンスを確認
for name, data in pme.user_data_registry.items():
    print(f"{name}: {data.__dict__}")

コンテキスト管理¶

class pme.context¶

PME の実行コンテキスト。内部使用が主ですが、カスタム関数の登録に使用できます。

add_global(key, value)¶

グローバル名前空間にカスタム関数または値を登録します。

パラメータ:

  • key (str) -- アクセス名

  • value -- 登録する関数または値

例:

def my_tool():
    bpy.ops.mesh.select_all(action='TOGGLE')

pme.context.add_global("toggle_select", my_tool)
globals: dict¶

グローバル名前空間辞書へのアクセス。

注釈

pme.context への直接アクセスは非推奨です。 新規コードは pme.execute() / pme.evaluate() を使用してください。

安定性レベル¶

レベル

説明

Experimental

v2.0.0 時点の暫定仕様。変更の可能性あり。

Internal

外部からの利用は非推奨。予告なく変更される可能性あり。

v2.0.0 では、すべての公開 API は Experimental です。 v2.1.0 以降で、利用実績を見て Stable に昇格するものを決定します。
