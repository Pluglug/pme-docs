# コンテキストメニュー

PME には、Blender の UI を右クリックした時に使えるコンテキストメニューがあります。

このメニューを使うと、今見えているボタンやオペレーターをそのままメニューへ追加したり、パスや値をコピーしたりできます。

## 基本的な使い方

### 右クリックから使う

いちばん確実なのは、追加したい UI 要素を **右クリック** して `Pie Menu Editor` を開く方法です。

右クリック経由では Blender が「今どの UI ボタンを指していたか」という情報を PME に渡してくれるため、取得精度が高くなります。

### ホットキーで直接使う

`User Interface` キーマップに `pme.button_add` などを登録すると、右クリックを経由せず直接追加することもできます。

この direct 実行は便利ですが、右クリックより制約が多くあります。  
現在の PME は次の順で取得を試します。

1. `button_pointer` / `button_prop`
2. `context.property`
3. `Copy Data Path` 相当の Blender API

そのため、UI の種類によって成功率が変わります。

### direct 実行で通りやすいもの

- サイドバーやツール設定などの通常のプロパティボタン
- `Scene`, `Object`, `Space Data` など、`C.` パスへ落としやすいもの
- `Preferences` の通常プロパティ
- `Addon Preferences`
- `Theme` の多くの色・数値プロパティ

### direct 実行で通りにくいもの

- 一時的に開く **Menu 系 UI**
- ポップアップ内の項目
- Blender が active button 情報を残さない UI
- `Copy Data Path` でも解決できない仮想的な UI 項目

特に **Menu 系は direct ホットキーより右クリックの方が安定** します。

## なぜ右クリックの方が強いのか

右クリックの button context menu は、Blender 側が hover 中の UI ボタン情報を明示的に組み立ててから呼び出します。

一方、`User Interface` キーマップから直接オペレーターを呼ぶ場合は、その瞬間に同じ情報が必ず残るとは限りません。

そのため、同じ UI 上の項目でも次のような差が出ます。

- 右クリックでは追加できる
- direct ホットキーでは「追加できません」になる

これは PME 側の制限だけでなく、Blender がその時点で何を context に残しているかにも依存します。

## うまく行かない時のおすすめ

1. まずは右クリックから試す
2. それで取れるなら、次に direct ホットキーを試す
3. direct で失敗したら `capture` ログを見る
4. Menu 系は right click 前提で使う

## ログについて

PME の Logger で `capture` カテゴリを有効にすると、次のような情報を確認できます。

- どの route で path を取れたか
- `context_path` / `data_path` のどちらが使われたか
- `property_path_unavailable` などの失敗理由

精度の改善はこのログを前提に進めています。  
もし「右クリックでは取れるのに direct では取れない」ケースがあれば、ログがあるとかなり追いやすくなります。
