# BlenderのKeymapとPME

PMEのホットキー設定で最初に戸惑うのは、多くの場合**キーの選び方**ではなく
**Keymapの選び方**です。Blenderのホットキーは「1つのキーに1つの機能」ではなく、
**文脈に応じて複数のKeymapから該当する項目を探す仕組み**になっています。

PMEはこの仕組みの外側に独自のショートカットを持つわけではありません。
PMEのホットキーはBlenderのKeymapの中に追加され、Blenderの検索ルールに
そのまま従って呼び出されます。

## 先に結論

- PMEのホットキーは、選んだKeymapに項目として追加されます(既存項目を上書きはしません)
- 同じキーが複数のKeymapに存在するのは普通のことです。モードや状況によって**使い分け**られています
- 実際に使われるのは、**今の文脈で一番具体的なKeymapにある項目**です
- PMEでは「どのキーを使うか」以上に、**どのKeymapに置くか**で挙動が決まります

::::{admonition} 「上書きする」より「積み重なっている」と考える
:class: tip

Blenderのホットキーは、1つの一覧を直接書き換えるよりも、**複数のKeymapが層になって
重なっている**と考えると理解しやすくなります。

たとえば同じ1つのキーに、**Edit Mesh 用のパイメニュー**を `Mesh` Keymapへ、
**Object Mode 用の別のパイメニュー**を `Object Mode` Keymapへ、それぞれ登録できます。
Blenderは今いるモードを見て、**その場面に合うほうを自動的に選んで**表示します。

同じキーが重なっていても「競合」ではなく「役割分担」として機能する、というのが
この仕組みの核心です。
:::::

## 3D Viewではどの順で探される？

3D Viewでキーを押したとき、実用上はおおむね次の順で候補が探されます。
**上にあるものほど具体的で、先に拾われます**。

1. モーダル操作中のオペレーター(移動・回転などの変形中に反応する層)
2. Screen Editing(ウィンドウ分割・ドラッグなどを扱う特殊な層)
3. アクティブツール(スカルプトブラシなど、いま選んでいるツール)
4. モード固有のKeymap(`Mesh`、`Sculpt`、`Object Mode` など)
5. エディタ固有のKeymap(`3D View` など)
6. Window(Blender全体)

「PMEが効かない」と感じるときの多くは、**登録に失敗したのではなく、
より上の層に置かれた別の項目が先にキーを受け取っている**だけです。

```{note}
キーが一致しても、**Pollの条件を満たさない項目はスキップされ**、次の候補が探されます。
これを利用して、たとえば「メッシュ選択中に有効なパイ」と「アーマチュア選択中に有効なパイ」を
同じキーに同居させ、選択対象に応じて自動で出し分ける、といった使い方もできます。
```

## PMEはこの中で何をしているか

PMEでKeymapを選んでホットキーを設定すると、**そのKeymapの中に「PMEメニューを呼び出す」
という項目が追加されるだけ**です。独自のショートカット層を別に持つわけではありません。

したがって、**どのKeymapに置くかが、そのパイメニューがどの場面で拾われるかを直接決めます**。

## まず何を選べばいい？

迷ったら、**元のBlender機能がどこに登録されているか**から逆引きするのが基本です。

1. Blender標準の似たショートカットや操作を調べる
2. その機能が属しているKeymapを確認する
3. PMEでもまず同じKeymapを選ぶ
4. 必要ならPollで条件を絞る

::::{admonition} PMEの `+` ボタンを使う
:class: hint

ホットキー欄の横にある `+` ボタンからは、**既存のBlender機能をキーマップから検索**できます。
「この操作は `Mesh` なのか `3D View` なのか `Window` なのか」を確認してから設定すると、
無理のない配置を選びやすくなります。
:::::

## よくある選び方

:::::{list-table}
:header-rows: 1
:widths: 34 22 44

* - やりたいこと
  - 選びたいKeymap
  - 理由
* - メッシュ編集中だけ使いたい
  - `Mesh`
  - Edit Mesh に入っている間だけ有効。他のモードでは発動しないので、意図しない場面で動く心配がない
* - オブジェクトモードでだけ使いたい
  - `Object Mode`
  - Object Mode 専用の操作として自然。`3D View` より具体的なので優先される
* - スカルプトブラシなどをパイで切り替えたい
  - `Sculpt`
  - 既存のスカルプト操作と同じ文脈で扱える
* - 3D View全体で(モードを問わず)使いたい
  - `3D View`
  - 3D Viewの文脈に限定しつつ、特定モードに縛られない
* - モードを**またいで**切り替えたい(モード切替パイなど)
  - `3D View`
  - モード固有Keymapに置くと「切り替え前のモードでしか呼べない」罠にはまります
* - 3D View以外(Outlinerなど)でも同じキーで呼びたい
  - `Window`
  - 最も広いが、より具体的なKeymapに同じキーがあると負けます
:::::

```{caution}
`Screen Editing` Keymap も選択肢に現れますが、**通常のPME利用では使いません**。
Blenderのウィンドウ操作を扱う特殊な層で、ほぼすべてのイベントを先取りするため、
他の機能と広く競合します。他のKeymapで実現できないか先に確認してください。
```

## 3D Viewで特にハマりやすい点

### `3D View` は「3D Viewのどこでも」ではない

`3D View` のKeymapが有効なのは、**ビューポートの描画領域の上にマウスがあるとき**だけです。
ヘッダー、ツールバー、Nパネル(サイドバー)の上ではキーが届かず、パイが出ません。

どの領域からも呼びたい場合は、`Window` への登録も合わせて検討してください。優先度は下がります。

### アクティブツールやモーダル操作が先にキーを取る

PMEに同じキーを登録しても、**いま選んでいるツール**(スカルプトブラシ、ナイフツールなど)や、
**操作の最中**(移動・回転などの変形中)はそちら側が先にキーを受け取ります。

「登録に失敗した」のではなく、**より具体的な層が先に反応している**だけです。
その場面でも呼びたい場合は、そのツール/モーダル側で空いているキーを選びます。

### モード切替パイは、モード固有Keymapに置かない

これが一番よくある落とし穴です。

「Object Mode 用のパイだから `Object Mode` Keymapに」と置いてしまうと、
**Edit Mode や Sculpt Mode に入ったあと、そのキーでパイを呼び戻せなくなります**。
モードを横断して呼びたいパイは `3D View` か `Window` に置きます。

### `Window` は便利だが万能ではない

`Window` は広く効きますが、より具体的なKeymapに同じキーの項目があれば**そちらに負けます**。
3D Viewで確実に動かしたいパイは、`Window` より `3D View` に置く方が素直です。

## 迷ったときの判断手順

1. その操作は「**どこで・どのモードで**使いたいか」を先に決める
2. Blender標準の似た操作がどのKeymapにいるかを `+` ボタンで確認する
3. まずは同じKeymapを選ぶ
4. モードを横断したい場合はモード固有Keymapを避ける
5. 条件で出し分けたいならPollを使う
6. `Screen Editing` は原則使わない
