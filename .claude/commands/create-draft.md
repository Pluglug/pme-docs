# Create Documentation Draft

指定されたセクションとトピックについて、ソースコードを調査してドキュメントのたたき台を作成します。

## Arguments

- `section`: ドキュメントのセクション（例: "editors", "getting_started", "reference"）
- `topic`: トピック名（例: "sticky_key", "basic_tutorial", "scripting_guide"）
- `file_name` (optional): ファイル名（省略時は `[topic].md`）

## 実行手順

### 1. セクションとトピックを理解

**セクションごとの目的:**
- `getting_started/` - 初心者向けチュートリアル、基本概念
- `editors/` - 各エディタの使い方、設定、機能説明
- `reference/` - 高度な機能、スクリプティング、用語集
- `support_community/` - FAQ、変更履歴、コントリビューション

### 2. 関連ソースコードを調査

Task toolのExploreエージェントを使用して：
- 関連する `ed_*.py` ファイル
- `operators.py` の関連オペレータ
- Helper/Utility モジュール

主要な情報を収集：
- UI要素とパラメータ
- ユーザー操作フロー
- 設定オプション
- 制限事項や注意点

### 3. たたき台を作成

`docs_ja/source/[section]/[file_name]` にたたき台を作成：

**基本テンプレート:**

```markdown
# [トピック名]

## 概要
[この機能の目的と基本的な用途]

## 前提知識
- [必要な前提知識がある場合]

## 基本的な使い方

### 手順1: [タイトル]
[手順の説明]

**TODO**: スクリーンショット

### 手順2: [タイトル]
[手順の説明]

## 設定項目

### [設定名1]
- **説明**: [何をする設定か]
- **デフォルト値**: [値]
- **使用例**: [具体例]

### [設定名2]
...

## 実践例

### 例1: [ユースケース]
[具体的な使用シーン]

**TODO**: GIF/動画

## Tips & Tricks
- [便利な使い方]
- [知っておくと良いこと]

## よくある質問

### Q: [質問]
A: [回答]

## 関連項目
- {ref}`関連ページ1`
- {ref}`関連ページ2`

## TODO
- [ ] スクリーンショット: [説明]
- [ ] 動画/GIF: [説明]
- [ ] 追加説明: [箇所]
```

### 4. 既存ドキュメントとの整合性

以下を確認：
- 用語の統一（既存ドキュメントと同じ用語を使用）
- スタイルの一貫性（見出しレベル、記法）
- 相互参照の追加（関連ページへのリンク）

### 5. レポート

作成結果を報告：
- 作成したファイル: `docs_ja/source/[section]/[file_name]`
- 調査したソースファイル: [リスト]
- 重要な実装箇所: [ファイル:行番号]
- 必要なビジュアル素材: [リスト]
- 追加調査が必要な項目: [リスト]

## Example

```bash
/create-draft editors sticky_key sticky_key_editor.md
```

または短縮形：

```bash
/create-draft editors sticky_key
```
（ファイル名は `sticky_key.md` に自動設定）

## Notes

- ユーザー向けの説明レベルを意識（技術的すぎない）
- 実装の詳細よりも「何ができるか」「どう使うか」に焦点
- 不明点や仮定はTODOとして明記
- スクリーンショット/動画が必要な箇所を具体的に指摘

## 作成後の推奨アクション

1. ユーザーがたたき台を確認・加筆
2. `/build-preview` でビルド確認
3. 完成したら `/translate-to-en [section]/[file_name]`
